Scholar.App Lexicon Documentation

Version 1.0.2 — April 27, 2025

Introduction

This document defines the record schemas for the Scholar.App decentralized scholarly network, built on the AT Protocol. Each record type is described in detail, including its fields, usage, and relationships to other types, along with example objects.

Record Definitions

üìÑ Paper (scholar.app.paper)

Represents the canonical metadata of a scholarly paper, pointing to its latest version and tracking citation activity.

  • title (string): The paper's title.
  • slug (string): URL-safe unique identifier for the paper.
  • abstract (string): Brief summary of the paper's content.
  • authors (array of objects): List of author information (DID, name, optional ORCID).
  • publishedAt (datetime): Date the paper was officially published.
  • latestVersionCid (string): CID pointing to the latest paperVersion record.
  • references (array of strings): CIDs of outgoing citation records.
  • citedBy (array of strings): CIDs of incoming citation records.
  • citationsCount (integer): Number of times cited.
  • createdAt (datetime): Timestamp of creation.
  • updatedAt (datetime): Timestamp of last update.

Example:

{
  "title": "A Study of Quantum Graphs",
  "slug": "quantum-graphs-study",
  "abstract": "An exploration into the applications of quantum graphs in network theory.",
  "authors": [
    { "did": "did:example:1234", "name": "Alice Quantum" }
  ],
  "publishedAt": "2025-04-01T12:00:00Z",
  "latestVersionCid": "bafybeicidv12345abcde",
  "references": [],
  "citedBy": [],
  "citationsCount": 0,
  "createdAt": "2025-04-01T12:00:00Z",
  "updatedAt": "2025-04-01T12:00:00Z"
}
üìú Paper Version (scholar.app.paperVersion)

Immutable snapshot of a specific version of a paper's content.

  • paperSlug (string): Associated paper slug.
  • versionNumber (integer): Version index.
  • bodyCid (string): CID of the full paper text (PDF/Markdown).
  • uploadedAt (datetime): Upload timestamp.
  • authorDids (array of strings): DIDs of contributing authors.
  • notes (string, optional): Version notes.

Example:

{
  "paperSlug": "quantum-graphs-study",
  "versionNumber": 1,
  "bodyCid": "bafybeibodyv6789zyxwv",
  "uploadedAt": "2025-04-01T12:00:00Z",
  "authorDids": ["did:example:1234"],
  "notes": "Initial draft"
}
üîó Citation (scholar.app.citation)

Links a paper version to an external or internal resource it cites.

  • citingVersionCid (string)
  • target (internal paper, DOI, URL, or BibTeX)
  • context (string, optional)
  • label (string, optional)
  • createdAt (datetime)

Example:

{
  "citingVersionCid": "bafybeibodyv6789zyxwv",
  "target": { "doi": "10.1038/s41586-020-2649-2" },
  "context": "Building on the findings from recent quantum network research.",
  "label": "Doe et al. (2020) Quantum Networks",
  "createdAt": "2025-04-01T12:10:00Z"
}
üìù Review (scholar.app.review)

Formal peer review of a paper version.

  • paperSlug (string)
  • versionNumber (integer)
  • reviewerDid (string)
  • decision (accept / revise / reject)
  • comments (string)
  • rating (number, optional)
  • createdAt (datetime)

Example:

{
  "paperSlug": "quantum-graphs-study",
  "versionNumber": 1,
  "reviewerDid": "did:example:reviewer789",
  "decision": "revise",
  "comments": "Interesting concept, but the experimental section needs improvement.",
  "rating": 3.5,
  "createdAt": "2025-04-05T10:30:00Z"
}
